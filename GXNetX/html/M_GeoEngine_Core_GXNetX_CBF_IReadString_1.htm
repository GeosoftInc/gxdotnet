Length of string to read, NULL will be added so the string length must be at least int+1.
Remarks
If the data source may be in byte order different from that required by the reader, you can add the source byte-order to the BF elelment type. The byte order will be swapped if required. For example, to write out a real number 3.5 with Most-Significant_Byte first (Mortorola) convention: WriteReal_BF(hBF,<define>BF_BYTEORDER_MSB</define>+<define>GS_REAL</define>,3.5). If a byte order is not specified, the source is assumed to be in the native byte order of the reading/writing computer.
